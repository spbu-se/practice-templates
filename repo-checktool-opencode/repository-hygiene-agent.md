# Субагент: Гигиена репозитория (Правила 9-10)

## Правила

### Правило 9: Правильный .gitignore
- Не должно быть результатов сборки (bin, obj, __pycache__, build/)
- В идеале не должно быть бинарных файлов
- Все файлы в .gitignore должны быть действительно лишними

### Правило 10: Отсутствие секретов
- Не должно быть ключей, токенов, паролей в репозитории
- Проверить Push protection в GitHub Settings
- Использовать .env файлы для секретов

## Анализ

1. Проверить .gitignore на наличие bin, obj, __pycache__, build/, dist/
2. Проверить наличие секретов (grep для API keys, passwords, tokens)
3. Проверить .github/workflows/ на написание секретов

## Результат

В результате надо вернуть табличку, каждая строчка которой соответствует одному правилу.
В табличке три столбца: номер правила, результат, описание причины.
Возможные результаты:
- `PASS` — гигиена соблюдена
- `FAIL` — нарушение с описанием
- `NA` — не применимо
