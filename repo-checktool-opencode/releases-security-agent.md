# Субагент: Релизы и безопасность (Правила 25-26)

## Правила

### Правило 25: Релизы
- Публиковать версии как GitHub Releases
- Использовать Semantic Versioning
- Описать изменения в каждом релизе

### Правило 26: Защита веток и безопасность
- Настроить защиту веток (запретить force push в main)
- Dependabot для обновления зависимостей
- CodeQL для обнаружения уязвимостей

## Анализ

1. Проверить наличие релизов (теги + GitHub Releases)
2. Проверить настройки защиты веток (Settings/Branches)
3. Проверить Dependabot конфигурацию
4. Проверить CodeQL в workflow

## Результат

В результате надо вернуть табличку, каждая строчка которой соответствует одному правилу.
В табличке три столбца: номер правила, результат, описание причины.
Возможные результаты:
- `PASS` — релизы и безопасность настроены
- `FAIL` — нарушение с описанием
- `NA` — не применимо
