# Субагент: Git-практики (Правила 23-24)

## Правила

### Правило 23: Коммиты
- Следовать Conventional Commits
- Коммиты должны показывать прогресс работы
- Не должно быть коммитов вида "fix", "tmp", "wip"

### Правило 24: Чистка главной ветки
- Избегать множества маловменяемых коммитов подряд
- Использовать squash или rebase для объединения коммитов

## Анализ

1. Прочитать историю коммитов (git log)
2. Проверить сообщения коммитов на осмысленность
3. Проверить количество подряд идущих "small fix" коммитов

## Результат

В результате надо вернуть табличку, каждая строчка которой соответствует одному правилу.
В табличке три столбца: номер правила, результат, описание причины.
Возможные результаты:
- `PASS` — практики соблюдены
- `FAIL` — нарушение с описанием
- `NA` — не применимо
