# Субагент: Качество кода (Правила 6, 8, 11, 20)

## Правила

### Правило 6: Форматтер/линтер (из CI/CD агента)
- Должен быть настроен форматтер/линтер
- Линтер запускается в CI

### Правило 8: Pre-commit hooks и IDE интеграция
- Имеет смысл настроить pre-commit hook для линтера
- Интегрировать линтер/форматтер с IDE (VS Code, Rider)

### Правило 11: Сторонние анализаторы
- CodeCov для тестового покрытия
- CodeFactor или Codacy как статический анализатор

### Правило 18: Правильное наименование
- Названия организаций, инструментов, технологий должны совпадать с официальными
- Примеры: TRIK Studio (не Trik Studio), RISC-V (не RISC-5)

### Правило 20: Соответствие стилю языка
- Python: PEP-8
- C++: Core Guidelines
- Другие языки: соответствующие стандарты

## Анализ

1. Найти конфигурацию линтера (.eslintrc, pyproject.toml, .editorconfig)
2. Найти pre-commit hooks (.pre-commit-config.yaml)
3. Проверить CodeFactor/Codacy/CodeCov интеграцию
4. Проверить соответствие стилю кода

## Результат

В результате надо вернуть табличку, каждая строчка которой соответствует одному правилу.
В табличке три столбца: номер правила, результат, описание причины.
Возможные результаты:
- `PASS` — качество кода соответствует
- `FAIL` — нарушение с описанием
- `NA` — не применимо
