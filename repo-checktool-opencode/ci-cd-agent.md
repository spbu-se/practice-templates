# Субагент: CI/CD системы (Правила 4-7)

## Правила

### Правило 4: Настройка CI-системы
- Для компилируемых языков: проверка сборки в каждой ветке и PR
- Для интерпретируемых: проверка качества кода и работоспособности
- Предпочтительно GitHub Actions, допустимы AppVeyor, CircleCI и др.

### Правило 5: Модульные тесты в CI
- Должны быть модульные тесты (кроме редких исключений)
- Тесты должны запускаться в CI

### Правило 6: Форматтер/линтер в CI
- Должен быть настроен форматтер и/или линтер
- Линтер должен запускаться в CI
- При ошибках линтера сборка должна падать
- Примеры: FSharpLint, flake8, Fantomas, black, eslint, golangci-lint

### Правило 7: Раздельный запуск линтера
- Запуск линтера может быть отдельной задачей в CI

## Анализ

1. Найти GitHub Actions workflow (.github/workflows/)
2. Проверить наличие jobs для сборки, тестов, линтера
3. Проверить тип языка проекта
4. Проверить наличие тестов (test/, __tests__/, *test*)

## Результат

В результате надо вернуть табличку, каждая строчка которой соответствует одному правилу.
В табличке три столбца: номер правила, результат, описание причины.
Возможные результаты:
- `PASS` — CI настроен корректно
- `FAIL` — нарушение с описанием
- `NA` — не применимо
